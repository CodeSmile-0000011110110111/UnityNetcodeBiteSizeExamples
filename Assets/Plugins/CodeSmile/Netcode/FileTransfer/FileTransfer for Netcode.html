— Why use FileTransfer? —
NetworkVariables and RPC calls can only send limited amounts of data at once. Combined traffic generated in the same network tick must not exceed 32 KB with Unity Transport for example.

To transfer large amounts of data (ie files) FileTransfer requests a transfer with the receiver who either accepts or denies. If accepted, a sequence of data packets (configurable size) are sent over time while the receiver puts them together in the right order.

When all data has been sent, the receiver verifies the data for correctnes and lets the sender know.

At any time during a transfer a client might disconnect or the user might decide to cancel the transfer. FileTransfer handles that, too.

— Features —
▶ Send data from: client to server / server to client
▶ Receiver can allow or deny a transfer request
▶ SHA1 checksum verification of received data
▶ NetworkFileTransfer.Singleton (file handling)
  Send Texture2D as PNG
  Send a file's raw contents
  More planned ...
▶ NetworkDataTransfer.Singleton (low-level)
  Send any byte[]
▶ OnRequest, OnProgress, OnFinished callbacks for:
  Client sending
  Client receiving
  Server sending
  Server receiving
